GDB

GDB is a debugger. GDB is an interactive tool.

Install gdb and ddd

when you compile make sure you use the -g option. The -g option tells the compiler to save the symbol table—that is, the list of memory addresses corresponding to your program’s variables and lines of code—within the generated executable.

gcc -g program.c -o programname
nasm -f elf64 -g -F stabs file.asm
gcc -g main.c

gdb ./a.out
ddd ./a.out

Command line

 gdb -ex 'break fun1' -ex 'break fun2' -ex 'break fun3' ./myprog
 gdb -ex 'break main' -ex 'info b' -ex 'set print pretty on' ./myprog

Configuration / Automation

GDB’s startup files are named .gdbinit by default (gdb executes file .gdbinit after running). You can have one in your home directory for general purposes and another in the directory containing a particular project for purposes specific to that project.

# Example .gdbinit
set pagination off
set logging file gdb.txt
set logging on
file a.out
b f
commands
bt
continue
end
info breakpoints
r
set logging off
quit

# run gdb on x first reading in commands from file z
gdb -command=z x

Errors

Segmentation faults occur when a running program attempts to access memory that it does not have permission to access. Typically the cause is an out-of-bounds array index or an errant pointer value.
